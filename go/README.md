# AntiSpamBotTG (Go)

Telegram-бот для автоматической модерации чатов. Работает как антифлуд: если пользователь присылает больше `RATE_LIMIT_COUNT` сообщений за `RATE_LIMIT_WINDOW_SECONDS`, бот мгновенно удаляет очередное сообщение и банит пользователя с отзывом (удалением) всех его недавних сообщений в чате — одним запросом к Telegram API (`RevokeMessages: true`).

Обработка асинхронная: входящие обновления от Telegram сразу кладутся в очередь и разбираются пулом воркеров, поэтому один медленный вызов Telegram API (удаление/бан) не блокирует приём остальных сообщений. При аномальном всплеске нагрузки, когда очередь переполняется, новые обновления отбрасываются с предупреждением в лог — вместо падения бота или неограниченного роста памяти.

Переписан с нуля на Go (изначальная версия была на Python/python-telegram-bot и использовала чёрный список слов) с прицелом на минимальное потребление ресурсов и работу 24/7 в Docker.

## Пример работы

```
Пользователь: привет
Пользователь: привет
Пользователь: привет
Пользователь: привет
Пользователь: привет
Пользователь: привет   <- 6-е сообщение за 3 секунды

Бот: [6-е сообщение удалено]
Бот: [пользователь забанен, его предыдущие 5 сообщений тоже удалены]
Бот: 🚫 Пользователь @spam_user123 забанен за флуд.
```

Лог бота:
```
2026-07-03 21:00:01 WARN обнаружен флуд, бан пользователя user_id=123 username=spam_user123 chat_id=-100123456
```

## Предварительные требования

- Docker
- Docker Compose (`docker compose`, входит в современный Docker Desktop / Docker Engine)
- Токен Telegram-бота (см. раздел ниже)

## Как получить токен бота и настроить права администратора

1. В Telegram откройте чат с [@BotFather](https://t.me/BotFather) и отправьте `/newbot`.
2. Следуйте инструкциям (имя, username бота) — BotFather пришлёт токен вида `789123:AAH...`. Это значение для `BOT_TOKEN`.
3. Добавьте бота в нужный чат/группу.
4. Откройте настройки группы → «Администраторы» → добавьте бота в администраторы.
5. Обязательно включите боту права **«Удаление сообщений»** (Delete messages) и **«Блокировка пользователей»** (Ban users) — без них бот не сможет удалять сообщения и банить флудеров.

## Деплой

1. Склонируйте репозиторий и перейдите в директорию с Go-версией:
   ```
   git clone https://github.com/Hristofor1234/AntiSpamBotTG.git
   cd AntiSpamBotTG/go
   ```
2. Скопируйте файл окружения и заполните своими значениями:
   ```
   cp .env.example .env
   nano .env
   ```
   Обязательно: `BOT_TOKEN` (из BotFather). Остальные переменные (`RATE_LIMIT_COUNT`, `RATE_LIMIT_WINDOW_SECONDS`, `WORKER_COUNT`, `QUEUE_SIZE`) необязательны — при отсутствии используются значения по умолчанию (5 сообщений / 3 секунды, 10 воркеров, очередь на 5000).
3. Запустите проект одной командой:
   ```
   docker compose up -d --build
   ```
4. Проверьте, что бот поднялся:
   ```
   docker compose logs -f bot
   ```
   В логе должна появиться строка о запуске бота с параметрами антифлуда.
5. Напишите боту `/start` в Telegram — должен прийти ответ с текущими настройками лимита.

## Команды управления

Логи:
```
docker compose logs -f bot
```

Перезапуск бота:
```
docker compose restart bot
```

Остановка:
```
docker compose down
```

Пересборка после изменения кода:
```
docker compose up -d --build
```

## Команды бота в Telegram

`/start` — приветственное сообщение с текущими параметрами антифлуда. Отдельных команд управления списком слов больше нет — бот работает полностью автоматически на основе частоты сообщений, без ручной модерации.

## Архитектура проекта

```
AntiSpamBotTG/go/
├── cmd/bot/main.go            # точка входа, связывает конфиг, лимитер, диспетчер и Telegram-бота
├── internal/config/           # загрузка и валидация конфигурации из окружения
├── internal/ratelimit/        # потокобезопасный счётчик сообщений на пользователя + периодическая очистка
├── internal/dispatcher/       # пул воркеров: очередь обновлений, бан флудеров через Telegram API
├── Dockerfile                  # multi-stage build, итоговый образ на alpine
└── docker-compose.yml          # единственный сервис bot (без внешней БД)
```

В этой версии бот не хранит состояние во внешней БД — счётчики сообщений живут только в памяти процесса и сбрасываются при перезапуске. Добавление PostgreSQL для белых/чёрных списков (постоянные исключения, ручная модерация) запланировано отдельным следующим шагом.
