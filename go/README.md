# AntiSpamBotTG (Go)

Telegram-бот для автоматической модерации чатов. Отслеживает все текстовые сообщения и мгновенно удаляет те, что содержат слова из чёрного списка. Список хранится в PostgreSQL, кэшируется в памяти и управляется администратором через команды бота.

Переписан с нуля на Go (изначальная версия была на Python/python-telegram-bot) с прицелом на минимальное потребление ресурсов и работу 24/7 в Docker.

## Пример работы

```
Пользователь: Купи криптовалюту прямо сейчас, переходи по ссылке!
Бот: [сообщение мгновенно удалено]

Лог бота:
2026-07-02 12:00:01 INFO удалено спам-сообщение username=spam_user123 matched_word=криптовалюту text_preview="Купи криптовалюту прямо сейчас, переходи по ссылке!"
```

```
Админ: /add крипта казино ставки
Бот: Добавлены: крипта, казино, ставки

Админ: /list
Бот: Слова:
крипта, казино, ставки
```

## Предварительные требования

- Docker
- Docker Compose (`docker compose`, входит в современный Docker Desktop / Docker Engine)
- Токен Telegram-бота (см. раздел ниже)

## Как получить токен бота и настроить права администратора

1. В Telegram откройте чат с [@BotFather](https://t.me/BotFather) и отправьте `/newbot`.
2. Следуйте инструкциям (имя, username бота) — BotFather пришлёт токен вида `789123:AAH...`. Это значение для `BOT_TOKEN`.
3. Добавьте бота в нужный чат/группу.
4. Откройте настройки группы → «Администраторы» → добавьте бота в администраторы.
5. Обязательно включите боту право **«Удаление сообщений»** (Delete messages) — без этого права бот не сможет удалять спам.

## Деплой

1. Склонируйте репозиторий и перейдите в его директорию:
   ```
   git clone https://github.com/Hristofor1234/AntiSpamBotTG.git
   cd AntiSpamBotTG
   ```
2. Скопируйте файл окружения и заполните своими значениями:
   ```
   cp .env.example .env
   ```
   Отредактируйте `.env`: укажите `BOT_TOKEN` (из BotFather), `DB_USER`, `DB_PASSWORD`, `DB_NAME`, `ALLOWED_USERS` (username администраторов через запятую). `DB_HOST` в docker-compose переопределяется автоматически на `db` — значение в `.env` можно оставить как есть.
3. Запустите проект одной командой:
   ```
   docker compose up -d --build
   ```
4. Проверьте, что бот поднялся:
   ```
   docker compose logs -f bot
   ```
   В логе должны появиться строки об успешной инициализации БД, загрузке кэша слов и запуске бота.
5. Напишите боту `/start` в Telegram — должен прийти ответ «Антиспам-бот активен...».
6. Добавьте первые спам-слова командой `/add слово1 слово2` (от имени пользователя из `ALLOWED_USERS`).

## Команды управления

Логи:
```
docker compose logs -f bot
```

Перезапуск бота:
```
docker compose restart bot
```

Остановка всего проекта:
```
docker compose down
```

Остановка с удалением данных БД:
```
docker compose down -v
```

Пересборка после изменения кода:
```
docker compose up -d --build
```

## Команды бота в Telegram

| Команда | Доступ | Описание |
|---|---|---|
| `/start` | все | приветственное сообщение |
| `/commands` | администраторы | список команд |
| `/add <слова>` | администраторы | добавить слова в чёрный список |
| `/remove <слова>` | администраторы | удалить слова из чёрного списка |
| `/list` | администраторы | показать чёрный список |

Администраторы определяются переменной окружения `ALLOWED_USERS` (список Telegram username через запятую).

## Архитектура проекта

```
AntiSpamBotTG/
├── cmd/bot/main.go          # точка входа
├── internal/config/         # загрузка и валидация конфигурации из окружения
├── internal/database/       # pgxpool, CRUD, in-memory кэш спам-слов (sync.RWMutex)
├── internal/handler/        # обработчики команд и фильтр спама
├── internal/middleware/     # проверка доступа (ACL) по ALLOWED_USERS
├── Dockerfile                # multi-stage build, итоговый образ на alpine
└── docker-compose.yml        # сервисы bot + db (postgres:15-alpine)
```

Спам-слова проверяются **только по кэшу в памяти** — запрос к БД не выполняется на каждое входящее сообщение. Кэш обновляется при старте и при выполнении `/add`/`/remove`.
